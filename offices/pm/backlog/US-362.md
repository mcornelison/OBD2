# US-362 — `detect_overlapping_drives(session, drive_id)` server compute helper

## Goal
As a server-side analytics consumer of raw `realtime_data`, I want a `detect_overlapping_drives(session, drive_id) -> list[int]` helper that returns the set of other drive_ids overlapping the given drive_id by any second of time-range intersection so that downstream compute paths (US-363 tripwire + US-364 backfill) have a SSOT overlap detector.

## Definition of Done (**at least 1 criterion required** so Ralph knows when complete)
- New function `detect_overlapping_drives(session, drive_id: int) -> list[int]` exists in server compute path (e.g., `src/server/analytics/overlap_detector.py` or per Ralph's structural call).
- Overlap threshold = **strict time-range intersection per second** (any second of overlap = match; per Atlas Refinements row 4); ε allowance only allowed with empirical justification documented in docstring.
- Function operates against `realtime_data` (raw signal authority per B-104 Step 1), not `drive_summary` derived rows.
- Unit tests cover: pairwise overlap (drives 23 + 24 from V0.27.18 IRL drill); no-overlap (drive 25 vs all neighbors); transitive overlap (3-way pairwise -- if drives 30/31/32 all overlap each other, all three appear in each call's result).
- Function is idempotent (no DB writes; pure query helper).
- Function documented with Google-style docstring per `specs/standards.md`.

## Conditional outcomes
- if the production `realtime_data` for drives 23+24 is no longer accessible (e.g., post-truncate), then unit tests must seed a fixture replicating the overlap window from F-107 description constants.
- if a 3-way transitive overlap surfaces in production data, then `detect_overlapping_drives(session, 30)` returns `[31, 32]` AND `detect_overlapping_drives(session, 31)` returns `[30, 32]` AND `detect_overlapping_drives(session, 32)` returns `[30, 31]`.
- if an empirical ε > 0 second allowance is later needed (e.g., to handle clock-jitter), then the change is a separate Story with empirical justification, not in-scope here.
