# US-367 — Backfill 2 ECU rows (prior + 2026-05-22 swap) + join verification

## Goal
As a server-side analytics consumer wanting per-drive ECU attribution, I want the historical ECU lineage backfilled into `vehicle_info` (prior ECU active before 2026-05-22 + new modified-EPROM ECU active from 2026-05-22 onward) so that any drive's `start_time_utc` can be joined to the ECU that was active at that time.

## Definition of Done (**at least 1 criterion required** so Ralph knows when complete)
- Backfill seeds exactly 2 rows in `vehicle_info`:
  - **Prior ECU**: `ecu_signature` = approximate-but-documented identifier (Spool sign-off on convention per F-108); `ecu_install_timestamp_utc` = CIO-provided approximate install date (source: `Eclipse 1998 Projects.gsheet` records OR earliest `realtime_data` row in chi-srv-01 if records unclear -- whichever is more reliable; document choice in commit body); `ecu_removal_timestamp_utc` = 2026-05-22 swap timestamp (precise per chain-merge history -- post-V0.27.18 IRL drill).
  - **New modified-EPROM ECU**: `ecu_signature` = identifier per F-108 ("ECMLink-V3-friendly-modified-EPROM-2026-05-22" or Spool-approved); `ecu_install_timestamp_utc` = 2026-05-22 swap timestamp (precise); `ecu_removal_timestamp_utc` = NULL (currently active).
- Data sources documented in commit body:
  - **Prior-ECU install timestamp**: CIO records (`Eclipse 1998 Projects.gsheet`) OR earliest `realtime_data.captured_at` -- approximate but documented per Atlas Refinements row 9.
  - **2026-05-22 swap timestamp**: precise per chain-merge history (post-V0.27.18 IRL drill 2026-05-22 ~13:30 CDT per CIO 2026-05-22 ratification).
- Spool sign-off on signature naming convention recorded BEFORE backfill (per F-108 Acceptance Criteria last bullet).
- Backfill executed via `stamp_ecu_swap` CLI (or a one-shot backfill script if `stamp_ecu_swap` doesn't fit the bootstrap path of stamping the FIRST row when no row currently exists).
- Join verification: `SELECT drive_summary.drive_id, vehicle_info.ecu_signature FROM drive_summary JOIN vehicle_info ON drive_summary.start_time_utc BETWEEN vehicle_info.ecu_install_timestamp_utc AND COALESCE(vehicle_info.ecu_removal_timestamp_utc, NOW())` correctly partitions drives 1-24 to prior ECU + drives 25+ to new ECU.

## Conditional outcomes
- if Spool has NOT signed off on the prior-ECU signature naming convention before backfill, then BLOCK the backfill + route an A2AL note to Spool for sign-off (per F-108 Acceptance Criteria last bullet).
- if `stamp_ecu_swap` cannot stamp the first row (bootstrap edge case -- no currently-active row exists per US-366 graceful error path), then a one-shot backfill script handles the first row insertion directly + subsequent stamps use `stamp_ecu_swap`.
- if join verification shows any drive 1-24 partitioning to new ECU (or any drive 25+ partitioning to prior ECU), then the install/removal timestamps are wrong; revise + re-backfill.
