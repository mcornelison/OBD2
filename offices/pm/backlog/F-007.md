## Description

Add tap-to-cycle / tap-to-toggle interaction to the existing HDMI dashboard (B-052). The original B-007 scope was "support the OSOYOO 3.5" HDMI capacitive touch screen (480x320) with a new `hdmi_touch` display mode" — that hardware path is **not** the current direction (CIO confirmed 2026-05-14: B-052 HDMI dashboard is the canonical full-canvas redesign; no separate `hdmi_touch` mode planned). Reframed scope: if the current HDMI dashboard hardware is capacitive-touch-capable, wire a single tap-to-cycle interaction (cycle through dashboard pages / toggle a forensic-debug overlay) without adding a new display mode.

## Acceptance Criteria (reframed)

- [ ] Confirm current HDMI display hardware exposes touch events via evdev or pygame
- [ ] Add tap-to-cycle handler to the existing HDMI dashboard renderer (one entry point, no new display mode)
- [ ] Touch handler is best-effort: absent touch hardware = no-op, dashboard works as before
- [ ] One synthetic test exercises the cycle-page-on-tap path with a mocked touch event

## Validation

- **IRL gate**: CIO taps screen during a live drive; dashboard advances one page per tap; no crash on absent-touch hardware

## Notes

Original 480x320 OSOYOO touchscreen plan dropped in favor of the full-canvas HDMI redesign (B-052). This item is now low-priority polish — keep in backlog for when B-052 lands and tap interaction becomes a useful gesture. If touch hardware isn't present at all on the deployed HDMI display, close this item then.
