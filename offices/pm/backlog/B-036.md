## Summary

Build the Chi-Srv-01 server as the project's analytics brain using a crawl/walk/run progression. Server code lives in `src/server/` within the OBD2v2 repo (design decision reversal from B-022's separate repo plan).

- **Crawl**: Server consumes simulator-generated fake data, produces analytics via CLI reports
- **Walk**: Accepts sync from Pi, auth middleware, stub AI
- **Run**: Real AI via Ollama/Spool, backup receiver, baseline calibration

## Key Design Decisions

- Server code in `src/server/` (NOT separate OBD2-Server repo)
- Fake data via export-import from existing simulator
- CLI reports over SSH as primary output (no web dashboard)
- Three analytics tiers: basic, advanced, AI
- Baselines always human-approved, never auto-applied
- FastAPI + MariaDB + SQLAlchemy + Ollama
