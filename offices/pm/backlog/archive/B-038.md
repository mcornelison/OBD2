## Summary

Pre-flight validator for `sprint.json` that catches authoring mistakes before Ralph burns a headless iteration. 8 checks, runs in <1 second. Wired into `/resize_sprint` skill and `ralph.sh` launcher.

## Location

`scripts/validate_sprint.py`

## Checks

| # | Check | Catches |
|---|---|---|
| 1 | JSON schema valid; required fields present with correct types | Authoring typo, missing field |
| 2 | Story IDs unique, match `US-\d+(-[a-z])?` | Duplicate or malformed ID |
| 3 | Size caps enforced: declared size vs filesToTouch/acceptance/diff | Story too large for its declared size |
| 4 | No acceptance criterion contains banned phrase | Weasel words that ruin headless execution |
| 5 | Every numeric literal in acceptance has matching groundingRefs | Invented values / magic numbers |
| 6 | Verification is non-empty array of executable commands | Vague "tests pass" with no command |
| 7 | Dependency graph acyclic, references only IDs in this sprint | Circular dep or typo |
| 8 | Size L stories must have pmSignOff non-empty | Oversized story without PM awareness |

## Error Output

Actionable, not cryptic:
```
[CHECK 4] stories[0]='US-147' acceptance[2] contains banned phrase 'handle edge cases'
          → rewrite as explicit condition, e.g.,
            'Input 220.0F produces CAUTION_HIGH, not DANGER'
```

## Acceptance Criteria

- Script exits 0 if all checks pass, 1 if any fail
- All 8 checks implemented
- Error messages include story ID, check number, and fix suggestion
- Runs in <1 second on full sprint file
- Integrated into `/resize_sprint` (blocks commit on failure)
- Integrated into `ralph.sh` (double-check before iteration)
