## Description

Implement tuning-driven alert thresholds for all Phase 2 parameters available only after ECMLink V3 and wideband O2 sensor are installed. These include real AFR (wideband), knock count/sum, injector duty cycle, true boost pressure, and ethanol content from the flex fuel sensor.

**Critical feature**: AFR thresholds MUST change dynamically based on ethanol content from the flex fuel sensor. The system interpolates linearly between gasoline and E85 thresholds based on actual ethanol percentage.

**Blocked**: Cannot be implemented or tested until ECMLink V3 is installed on the vehicle. Stories are fully specified so Ralph can execute immediately once unblocked.

**Source of truth**: All thresholds from Spool (Tuning SME). PM Rule 7 applies.

## User Stories

### US-113: Implement Wideband AFR Alert Thresholds — Pump Gas (93 Octane)

**As a** driver on pump gas, **I want** air-fuel ratio alerts based on operating condition **so that** I can detect dangerous lean conditions under boost.

**IMPORTANT — Alert Level Definitions (applies to US-113 and US-114):**
- **Target**: The ideal sweet spot the tune aims for. Narrower than Normal.
- **Normal**: Any AFR value NOT in Caution or Danger range. No alert triggered.
- **Caution**: Specific numerical bounds triggering yellow alert.
- **Danger**: Specific numerical bounds triggering red alert.

**Example (Idle, Pump Gas)**: Target = 14.7:1, but any value between 14.0 and 15.5 is Normal (no alert). Only below 14.0 or above 15.5 triggers Caution. Only below 13.0 or above 16.0 triggers Danger.

**Acceptance Criteria:**
- [ ] AFR thresholds vary by operating condition (idle, cruise, WOT):
  - **Idle**:
    - Target: 14.7:1 (stoichiometric)
    - Normal: 14.0 to 15.5 (inclusive)
    - Caution: < 14.0 or > 15.5
    - Danger: < 13.0 or > 16.0
  - **Cruise (part throttle)**:
    - Target: 14.5-15.0:1
    - Normal: 13.5 to 15.5 (inclusive)
    - Caution: < 13.5 or > 15.5
    - Danger: < 13.0 or > 16.5
  - **WOT (full boost)**:
    - Target: 11.0-11.5:1
    - Normal: < 11.5 (rich of target is safe at WOT — no upper Normal bound)
    - Caution: 11.5-12.0:1 (slightly lean)
    - Danger: **> 12.5:1 (LEAN UNDER BOOST — STOP)**
- [ ] Operating condition determined from TPS and RPM (idle < 5% TPS, WOT > 80% TPS, cruise = between)
- [ ] Thresholds sourced from config
- [ ] WOT lean danger alert is highest priority — audible + visual
- [ ] Tests validate each condition/threshold combination, including Normal range (e.g., 14.3 at idle = Normal, not Caution)

**Rationale (from Spool):** Earlier draft did not explicitly define "Normal" range, leaving ambiguity between Target and Caution. Clarified 2026-04-12 by Spool. Rich-of-target at WOT is intentionally safe (running 10.8:1 at WOT is rich but not dangerous — only lean AFR under boost is dangerous).

### US-114: Implement Wideband AFR Alert Thresholds — E85

**As a** driver on E85, **I want** E85-specific AFR alerts **so that** thresholds match the different stoichiometric ratio of ethanol.

**Alert Level Definitions**: Same as US-113 (Target = ideal, Normal = not Caution and not Danger, Caution and Danger are explicit numerical bounds).

**Acceptance Criteria:**
- [ ] E85 AFR thresholds vary by operating condition:
  - **Idle**:
    - Target: 9.8:1 (E85 stoichiometric)
    - Normal: 9.0 to 10.5 (inclusive)
    - Caution: < 9.0 or > 10.5
    - Danger: < 8.5 or > 11.0
  - **Cruise**:
    - Target: 9.8-10.2:1
    - Normal: 9.0 to 10.5 (inclusive)
    - Caution: < 9.0 or > 10.5
    - Danger: < 8.5 or > 11.5
  - **WOT (full boost)**:
    - Target: 7.5-8.0:1 (Lambda 0.68-0.72)
    - Normal: < 8.0 (rich of target is safe at WOT)
    - Caution: 8.0-8.5:1
    - Danger: **> 8.8:1 (LEAN UNDER BOOST — STOP)**
- [ ] E85 thresholds stored separately from gas thresholds in config
- [ ] Tests validate each condition/threshold combination for E85, including Normal range

### US-115: Implement Ethanol-Based AFR Threshold Interpolation

**As a** driver with a partial E85/gas blend, **I want** AFR thresholds to adjust automatically based on actual ethanol content **so that** alerts are accurate for any fuel mix.

**CRITICAL NOTE**: AFR thresholds MUST change based on ethanol content. The flex fuel sensor reports ethanol percentage. At 0% ethanol (pure gas), use gas thresholds. At 85% ethanol (E85), use E85 thresholds. In between, interpolate linearly.

**Acceptance Criteria:**
- [ ] Interpolation formula implemented:
  ```
  ethanol_fraction = ethanol_percent / 100
  gas_fraction = 1 - ethanol_fraction
  interpolated_value = (gas_fraction × gas_threshold) + (ethanol_fraction × e85_threshold)
  ```
- [ ] Interpolation applied to ALL AFR thresholds (target, caution, danger) for ALL conditions (idle, cruise, WOT)
- [ ] Ethanol content read from flex fuel sensor in real-time
- [ ] Tests validate interpolation with Spool's worked example:
  ```
  Input: ethanol_content = 62%, operating_condition = WOT
  
  Gas WOT target midpoint: 11.25:1
  E85 WOT target midpoint: 7.75:1
  Gas WOT danger (lean): 12.5:1
  E85 WOT danger (lean): 8.8:1
  
  Interpolated WOT target midpoint:
    (0.38 × 11.25) + (0.62 × 7.75) = 4.275 + 4.805 = 9.08:1
  
  Interpolated WOT danger threshold:
    (0.38 × 12.5) + (0.62 × 8.8) = 4.75 + 5.456 = 10.21:1
  
  Interpolated WOT target range:
    Low: (0.38 × 11.0) + (0.62 × 7.5) = 8.83:1
    High: (0.38 × 11.5) + (0.62 × 8.0) = 9.33:1
  ```
- [ ] With measured AFR of 10.2:1 at 62% ethanol WOT:
  - Status = CAUTION — AFR is at the lean boundary (danger threshold is 10.21:1)
  - Alert: "AFR 10.2:1 is within 0.01 of danger threshold (10.21:1) for 62% ethanol blend. Tune may need more fuel at WOT for mid-blend ethanol."
- [ ] Tests validate boundary: 50% ethanol → WOT danger = midpoint of 12.5 and 8.8 = 10.65:1

### US-116: Implement Knock Count Alert Thresholds

**As a** driver, **I want** knock event counting and alerts **so that** I can detect detonation before it damages the engine.

**Acceptance Criteria:**
- [ ] Knock count thresholds (from ECMLink knock sensor):
  - **Normal**: 0 — No knock detected. Happy engine.
  - **Caution**: 1-3 per drive — Some knock. Log RPM, load, timing, AFR at each event. Review pattern. Could be false knock (heat shield rattle, belt chirp).
  - **Danger**: > 3 per drive OR > 1 per WOT pull — **Active detonation. Do not increase boost. Review tune at affected RPM/load cells.**
- [ ] Each knock event logs contextual data: RPM, load, timing_advance, AFR, boost, TPS, IAT, coolant
- [ ] Per-WOT-pull knock counting (WOT pull = TPS > 80% sustained for > 2 seconds)
- [ ] Thresholds sourced from config
- [ ] Tests validate per-drive and per-WOT-pull counting

**Developer Reference — Knock Analysis Example (from Spool):**
```
Event 1 (under boost) — LIKELY REAL KNOCK:
  RPM: 4200, Load: 75%, Boost: 13.5 psi, TPS: 85%
  AFR: 11.8:1 — slightly lean for WOT (target: 11.0-11.5:1)
  Timing pulled: 3 degrees (significant ECU response)
  Assessment: High load + slightly lean AFR = plausible detonation.

Event 2 (light cruise) — LIKELY FALSE KNOCK:
  RPM: 2100, Load: 25%, Boost: vacuum (-15 inHg), TPS: 12%
  Real detonation at 2100 RPM / 25% load / vacuum is extremely unlikely.
  Assessment: Probable false trigger — heat shield rattle, exhaust tick, or road vibration.
```

### US-117: Implement Knock Sum Alert Thresholds

**As a** driver, **I want** cumulative knock energy tracking **so that** I can detect persistent knock problems.

**Acceptance Criteria:**
- [ ] Knock sum thresholds (cumulative per drive):
  - **Normal**: 0 — Perfect.
  - **Caution**: 1-5 — Minor knock energy accumulated. Review.
  - **Danger**: > 10 — **Significant knock energy. The tune needs attention at the affected operating points.**
- [ ] Knock sum resets per drive
- [ ] Thresholds sourced from config

### US-118: Implement Injector Duty Cycle (IDC) Alert Thresholds

**As a** driver, **I want** fuel system headroom alerts **so that** I know when injectors are approaching their limit.

**Acceptance Criteria:**
- [ ] IDC thresholds:
  - **Normal**: 0-75% — Plenty of headroom.
  - **Caution**: 75-85% — Getting warm. Injectors are working hard.
  - **Danger**: > 85% — **Fuel system is at its limit. Lean condition imminent at higher RPM/load. Upgrade injectors before increasing boost.**
  - **Static flow limit**: > 95% — **Injectors are maxed out. Engine WILL go lean above this point.**
- [ ] Peak IDC tracked per drive
- [ ] Thresholds sourced from config
- [ ] Tests validate all 4 levels including the 95% static limit

### US-119: Implement Boost Pressure Alert Thresholds

**As a** driver, **I want** boost pressure alerts including spike and creep detection **so that** I can catch wastegate and turbo issues.

**Acceptance Criteria:**
- [ ] Boost pressure thresholds (stock turbo, ECMLink MAP sensor):
  - **Normal**: 10-14 psi — Target boost range.
  - **Caution**: 14-15 psi — Approaching turbo efficiency limit.
  - **Danger**: > 15 psi — **Stock turbo is out of its efficiency range. Compressor surge risk. EGTs climbing.**
- [ ] Boost spike detection: > 3 psi over target momentarily — **Wastegate control issue. Investigate duty cycle or wastegate actuator.**
- [ ] Boost creep detection: Boost exceeds target and won't come down — **Exhaust flow is overpowering the wastegate. Needs larger wastegate or exhaust restriction.**
- [ ] Spike vs creep distinguished by duration (spike = < 2 sec, creep = sustained)
- [ ] Boost target configurable (may change with tune adjustments)
- [ ] Thresholds sourced from config
- [ ] Tests validate normal/caution/danger plus spike and creep detection

### US-120: Implement Ethanol Content Alert Thresholds

**As a** driver, **I want** ethanol content monitoring **so that** I know what fuel is actually in my tank.

**Acceptance Criteria:**
- [ ] Ethanol content thresholds (from flex fuel sensor):
  - **Expected E85**: 70-85% ethanol — Normal. "E85" at the pump varies widely.
  - **Low ethanol**: 50-70% — Pump may be selling diluted E85. Tune still safe (ECMLink interpolates). Log and inform driver.
  - **Unexpected gas**: < 30% — Someone put gas in the tank. ECMLink will switch to gas map. Inform driver.
  - **Unexpected high**: > 85% — Racing E85 or very fresh batch. Tune is fine. Note it.
- [ ] Thresholds sourced from config
- [ ] Alert messages are informational (no danger — ECMLink handles fuel map switching)
- [ ] Tests validate all 4 ranges
