# BL-002: Ralph Workspace Sandbox Restricts Access to Repo Root

| Field        | Value                     |
|--------------|---------------------------|
| Severity     | Critical                  |
| Status       | Active                    |
| Blocking     | All sprint stories (US-DBI-001/002/003, US-101/102/103, US-104/105/106, US-OSC-*) |
| Waiting On   | CIO to relaunch Ralph from repo root (Z:\o\OBD2v2) instead of offices/ subdirectory |
| Created      | 2026-04-11                |

## Description

Ralph agents launched from `Z:\o\OBD2v2\offices\` are sandboxed to only access files within `offices/`. The Claude Code security sandbox enforces this: any attempt to read, write, or list files in `src/`, `tests/`, `scripts/`, `specs/`, or any other repo-root directory is blocked.

Every remaining story in the sprint requires access to files outside `offices/`:
- **US-DBI-001/002/003** (Database Verify): `scripts/verify_database.py`, `src/obd/database.py`, `tests/test_verify_database.py`
- **US-101/102/103** (Simulate DB Validation): `tests/test_simulate_db_validation.py`, `src/obd/orchestrator.py`
- **US-104/105/106** (Ollama Cleanup): `specs/architecture.md`, `src/ai/ollama.py`, `src/ai/exceptions.py`
- **US-OSC-015 through US-OSC-020** (Orchestration): `tests/test_orchestrator_integration.py`, `src/obd/orchestrator.py`, `deploy/`

## Impact

**Full sprint block.** Zero stories can be completed by any agent launched from `offices/`. All 29 remaining stories (of 30 total) are blocked. Only US-DBI-004 (already complete) could have been done from within offices/ since it was pre-verified.

## Attempted Solutions

1. Agent 1 (Rex) attempted to read `specs/standards.md` via absolute path — blocked by sandbox
2. Agent 1 attempted `ls Z:/o/OBD2v2/src/` — blocked by sandbox
3. The blocker was previously noted in `ralph_agents.json` on 2026-04-09 but not formally filed

## Proposed Resolution

**Relaunch Ralph from the repo root directory:**
```bash
cd Z:\o\OBD2v2
./offices/ralph/ralph.sh 10
```

The `ralph.sh` script or the CIO's launch command needs to set the working directory to `Z:\o\OBD2v2` (the git repo root), not `Z:\o\OBD2v2\offices\`. All Ralph agent files (`agent.md`, `stories.json`, etc.) are referenced via `offices/ralph/` paths from the repo root, which is already the convention in CLAUDE.md and the stories.
