# BL-001: Ralph Agent Sandboxed to offices/ — Cannot Access Repo Root

| Field        | Value                     |
|--------------|---------------------------|
| Severity     | Critical                  |
| Status       | Active                    |
| Blocking     | All sprint stories (US-DBI-001, US-101, US-104, US-OSC-015, and all dependents) |
| Waiting On   | CIO to relaunch Ralph from repo root (Z:\o\OBD2v2) instead of offices/ subdirectory |
| Created      | 2026-04-09                |

## Description

Ralph agent (Rex, Agent 1) was launched with working directory `Z:\o\OBD2v2\offices` instead of `Z:\o\OBD2v2` (the repository root). Claude Code sandboxes file access to the allowed working directories, which means the agent cannot:

- Read or write files in `src/`, `scripts/`, `tests/`, `specs/`, `docs/`, `deploy/`
- Run `pytest`, `make lint`, or any quality checks
- Create or modify implementation files

## Impact

**All 29 remaining stories in Sprint 2026-04-01 are blocked.** Every story requires at least one of:
1. Creating/modifying files outside `offices/` (e.g., `scripts/verify_database.py`, `src/` modules)
2. Running tests (`pytest tests/`) from the repo root
3. Reading specs (`specs/standards.md`) for coding conventions

No implementation work can proceed until this is resolved.

## Attempted Solutions

1. Tried `Read` tool with full paths to `specs/standards.md` — permission denied
2. Tried `Bash` with `cd Z:/o/OBD2v2` — blocked by sandbox
3. Tried `Bash` with `ls`, `head`, `find` on repo root — all blocked
4. Tried `Glob` with explicit path to repo root — permission requested but not granted

## Proposed Resolution

Relaunch Ralph from the repository root directory:

```bash
cd Z:\o\OBD2v2
./offices/ralph/ralph.sh 10
```

Or ensure the Claude Code session's working directory is set to `Z:\o\OBD2v2` (not the `offices/` subdirectory).
