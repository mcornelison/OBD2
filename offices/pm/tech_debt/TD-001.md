## Description

The `TestDataManager` class in `tests/test_utils.py` has an `__init__` constructor. Pytest attempts to collect it as a test class because of the `Test` prefix, then emits a warning because it can't instantiate it (due to the constructor arguments).

## Why It Was Accepted

The class provides useful test data management functionality. Renaming it would require updating all references.

## Risk If Not Addressed

- No functional risk -- tests still run and pass
- Minor noise in pytest output (warning message)
- Could mask other legitimate warnings

## Remediation Plan

Either rename the class to `DataManager` (drop `Test` prefix) or add `__test__ = False` class attribute to suppress collection. Low effort, can be bundled with any cleanup task.

## Closure (US-270 / Sprint 23 / Rex Session 146 / 2026-05-03)

**Option (b) shipped**: `__test__ = False` class attribute on `TestDataManager` at `tests/test_utils.py:498`. Single-line addition; zero caller impact (consumer tests in `tests/test_test_utils.py::TestTestDataManager` continue to pass; no rename, no API change).

**Pre-fix runtime-validation gate**: `pytest tests/test_utils.py -v` emitted `PytestCollectionWarning: cannot collect test class 'TestDataManager' because it has a __init__ constructor (from: tests/test_utils.py)` -- 1 warning in 7.83s.

**Post-fix verification**:
- `pytest tests/test_utils.py -v` -- no warnings (collected 0 items in 9.48s; warning summary block gone entirely).
- `pytest tests/test_test_utils.py -v` -- 40/40 passed in 9.36s including the 3 `TestTestDataManager::test_testDataManager_*` consumer tests.
- Full fast-suite re-grep `pytest tests/ -m 'not slow' -q 2>&1 | grep -i 'TestDataManager'` returns no matches (the warning is gone from the canonical project-wide test run, satisfying TD remediation goal "no more pytest warning every test run").

**Verified by**: TestDataManager is the only class in the repository with the `Test` prefix that needed `__test__ = False` (pre-flight `rg TestDataManager tests/ src/` returned exactly one class declaration at `tests/test_utils.py:486`; the `TestTestDataManager` test class in `tests/test_test_utils.py:558` is a real pytest class and is correctly collected without `__test__ = False`).
