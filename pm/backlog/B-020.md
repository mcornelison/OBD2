## Description

The config file `src/obd_config.json` has two values that don't match the actual hardware and project decisions:

1. `display.width: 240, display.height: 240` -- should be `480, 320` (OSOYOO 3.5" HDMI display)
2. `dataRetention.realtimeDataDays: 7` -- should be `365` (changed in US-TD-003 code defaults but not in config file)

## Acceptance Criteria

- [ ] `display.width` set to `480` in `src/obd_config.json`
- [ ] `display.height` set to `320` in `src/obd_config.json`
- [ ] `dataRetention.realtimeDataDays` set to `365` in `src/obd_config.json`
- [ ] All tests still pass after config change
- [ ] Typecheck passes

## Validation Script Requirements

- **Input**: Read `src/obd_config.json`
- **Expected Output**: display dimensions are 480x320, realtimeDataDays is 365

## Notes

Small change but impacts Pi deployment (B-012) -- the display won't render correctly with wrong dimensions. Should be done before deploying to Pi.
